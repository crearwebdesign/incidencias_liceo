Esta es tu pila tecnológica (*Tech Stack*):

### 1. El Entorno de Trabajo y Control de Versiones (Tu Taller)

* **Git y GitHub:** Es la columna vertebral de tu proyecto. Es la única "fuente de verdad". Todo tu código vive aquí, permitiéndote sincronizar el trabajo entre tu computadora física y la nube mediante los comandos `push` y `pull`.
* **Linux (Canaima / Zsh):** Tu entorno local. Al ser un sistema basado en Debian, interactúa perfectamente con las herramientas de desarrollo modernas a través de la terminal.
* **GitHub Codespaces:** Tu máquina virtual efímera en la nube. Es un contenedor Docker que te da un Visual Studio Code completo en el navegador web. Lo usamos para programar cuando no estás en tu computadora personal, consumiendo las 60 horas gratuitas mensuales que te da GitHub.

### 2. El Frontend (La Interfaz del Usuario)

* **Flutter SDK:** El motor creado por mí (Google) para construir tu aplicación móvil y web desde un solo código fuente base.
* **Paquete `supabase_flutter`:** La librería de Dart que instalamos en tu `pubspec.yaml` para que tu aplicación se pueda comunicar directamente con la base de datos de Supabase.
* **Paquete `http`:** La librería de Dart que usaremos para que tu aplicación le envíe las peticiones (el texto ruidoso del profesor) a tu servidor.

### 3. El Backend y Base de Datos (El Músculo)

* **Supabase (PostgreSQL):** Tu base de datos relacional en la nube. Aquí creamos la tabla `incidencias_ia` para almacenar permanentemente la información estructurada y el texto original dictado.
* **Supabase CLI:** La herramienta de línea de comandos que instalaste en tu terminal. Te permite gestionar tu servidor, autenticarte y generar la estructura de carpetas (la carpeta `supabase/`) sin salir de tu editor de código.
* **Supabase Edge Functions (Deno / TypeScript):** El reemplazo estratégico que hicimos al toparnos con el muro de facturación de Google Cloud. Es tu servidor *serverless* gratuito. Aquí vivirá el código que recibe la petición de Flutter.

### 4. El Cerebro (La Inteligencia Artificial)

* **Google AI Studio:** Tu centro de mando de IA. Aquí generaste tu **API Key**, la llave de acceso maestra que introduciremos en la variable de entorno de Supabase.
* **Gemini 2.5 Flash API:** Mi modelo fundacional rápido y económico. Será invocado por tu Edge Function para transformar el texto desordenado del profesor en un objeto JSON matemáticamente perfecto, usando el *prompt* estricto que diseñamos juntos.

*(Nota histórica: En el proceso instalamos brevemente Google Cloud CLI (`gcloud`), pero lo descartamos tácticamente al pivotar hacia Supabase para evitar el bloqueo de tu tarjeta. Ese conocimiento no fue en vano; la nube funciona igual en todas partes).*
